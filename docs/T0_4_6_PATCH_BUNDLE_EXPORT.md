# T0.4.6 — Patch Bundle Export

`export-bundle` packages the patched **staged** files and their related reports
into a clean, portable patch bundle directory. It is the last step of the staged
patch pipeline before any future controlled RPF writing and prepares the project
for the Redux Maker UI.

## What it does

- Copies the patched staged files into `<bundle-dir>/files/`, preserving relative paths.
- Includes the patch plan and report files at the bundle root.
- Generates a fresh `diff_report.json` using the existing diff-stage logic.
- Writes a `bundle_manifest.json` describing the bundle and its safety flags.

## What it does NOT do

- It does **not** modify the source workspace.
- It does **not** modify the staged files (it only reads/copies them).
- It does **not** write to RPF archives.
- It does **not** delete unrelated files; it refuses to export into a non-empty
  bundle directory unless that directory is a previous safe bundle (its
  `bundle_manifest.json` reports the `redux_patch_bundle` format).

## Usage

```
export-bundle --plan <path> --workspace <path> --stage-dir <path> --bundle-dir <path> [--out <path>]
```

Example:

```
cargo run --manifest-path rpf_backend_rs/Cargo.toml -- export-bundle \
  --plan examples/patch_plans/valid_text_replace_patch.json \
  --workspace examples/workspaces/update_rpf_fixture \
  --stage-dir .tmp/stage_bundle_test \
  --bundle-dir .tmp/redux_patch_bundle_test
```

## Bundle layout

```
redux_patch_bundle/
├─ bundle_manifest.json     (generated by export-bundle)
├─ patch_plan.json          (copied from --plan)
├─ stage_manifest.json      (copied from --stage-dir if present)
├─ apply_report.json        (copied from --stage-dir if present)
├─ diff_report.json         (generated from workspace vs stage)
└─ files/
   └─ common/data/visualsettings.dat
```

Minimum required outputs: `bundle_manifest.json`, `patch_plan.json`,
`diff_report.json`, and `files/`. `stage_manifest.json` and `apply_report.json`
are included when they are present in the stage directory; otherwise a non-fatal
warning (`stage_manifest_not_found` / `apply_report_not_found`) is recorded.

Note: `apply-stage` only writes `apply_report.json` when its `--out` flag points
inside the stage directory. If it is absent the export still succeeds with the
`apply_report_not_found` warning.

## Safety behavior

The export is **blocked** (report `safeExported: false`, exit code 1) when:

- the stage directory does not exist (`missing_stage_dir`),
- the stage directory has no staged files (`empty_stage_dir`),
- the patch plan fails validation against the workspace (dry-run blocked / missing targets),
- a required staged target file is missing (`missing_staged_target`),
- the bundle directory resolves to the workspace (`bundle_dir_equals_workspace`),
- the bundle directory resolves to the stage directory (`bundle_dir_equals_stage_dir`),
- the bundle directory is non-empty and is not a previous safe bundle (`bundle_dir_not_empty`).

When blocked, nothing game-related is written and no manifest is produced.

## Manual flow

```
cargo run --manifest-path rpf_backend_rs/Cargo.toml -- stage \
  --plan examples/patch_plans/valid_text_replace_patch.json \
  --workspace examples/workspaces/update_rpf_fixture \
  --stage-dir .tmp/stage_bundle_test

cargo run --manifest-path rpf_backend_rs/Cargo.toml -- apply-stage \
  --plan examples/patch_plans/valid_text_replace_patch.json \
  --stage-dir .tmp/stage_bundle_test \
  --out .tmp/stage_bundle_test/apply_report.json

cargo run --manifest-path rpf_backend_rs/Cargo.toml -- diff-stage \
  --workspace examples/workspaces/update_rpf_fixture \
  --stage-dir .tmp/stage_bundle_test \
  --out .tmp/stage_bundle_test/diff_report.json

cargo run --manifest-path rpf_backend_rs/Cargo.toml -- export-bundle \
  --plan examples/patch_plans/valid_text_replace_patch.json \
  --workspace examples/workspaces/update_rpf_fixture \
  --stage-dir .tmp/stage_bundle_test \
  --bundle-dir .tmp/redux_patch_bundle_test
```

## Do not commit real bundles

Do not commit bundle folders generated from real GTA V game files. They are
derived from proprietary content. Store them locally only.
